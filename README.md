# go-worldcup-journal

## 项目简介

`go-worldcup-journal` 是一个用于归档和发布多个独立 HTML 页面的仓库。  
本项目不针对任何特定网站或域名，仅作为静态页面内容的存储与展示空间。  
所有页面均为独立编写，可单独访问，适用于内容归档、示例展示或轻量级信息发布。

## 目录说明

```
go-worldcup-journal/
├── pages/          # 存放所有独立 HTML 页面
├── assets/         # 公共资源文件（如样式、脚本、图片等）
├── index.html      # 仓库入口页（可选）
└── README.md       # 本文件
```

- `pages/`：每个 HTML 文件为一个独立页面，文件名建议使用英文或数字，便于直接访问。
- `assets/`：存放页面引用的外部资源，建议按类型（css/、js/、img/）子目录组织。
- `index.html`：可选入口文件，可提供页面列表或导航链接。

## 页面归档说明

本仓库中的 HTML 页面均为独立归档，具有以下特点：

- 每个页面完整自包含（或引用 `assets/` 中的资源）。
- 页面内容不依赖外部服务，可离线查看。
- 页面之间无强制关联，可单独使用或通过入口页索引。
- 归档内容以通用信息为主，不包含具体商业或营销元素。

## 维护说明

- 添加新页面：在 `pages/` 目录下新建 HTML 文件，并确保资源引用路径正确。
- 更新页面：直接修改对应文件，保持格式一致。
- 删除页面：移除对应文件，并更新入口页（如有）。
- 建议在提交时附带简要说明，方便追溯变更。

## 使用方式

1. 克隆或下载本仓库。
2. 直接在浏览器中打开 `pages/` 下的任意 HTML 文件，或通过 `index.html` 浏览。
3. 也可将本仓库部署到 GitHub Pages 或其他静态托管服务，实现在线访问。
